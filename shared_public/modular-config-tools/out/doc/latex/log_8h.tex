\subsection*{Off}

\subsection*{Logging is disabled on this log level.}

\subsection*{Fatal}

\subsection*{Fatal is used to log all unhandled exceptions and handled errors where it is not recommend to continue program}

\subsection*{execution.}

\subsection*{This is typically logged inside a catch block at the boundary of your application.}

\subsection*{Error}

\subsection*{Errors are handled \textquotesingle{}exceptions\textquotesingle{} above warnings.}

\subsection*{While program operation can safely continue on warnings, errors indicate a critical program state where a master}

\subsection*{software part has to consider weather to continue or to stop program execution.}

\subsection*{Warning}

\subsection*{Warning is often used for handled \textquotesingle{}exceptions\textquotesingle{} or other important log events.}

\subsection*{For example, if your application requires a configuration setting but has a default in case the setting is missing,}

\subsection*{then the Warning level should be used to log the missing configuration setting.}

\subsection*{Information}

\subsection*{The Information level is typically used to output information that is useful to the running and management of your}

\subsection*{system. Information would also be the level used to log Entry and Exit points in key areas of your application.}

\subsection*{However, you may choose to add more entry and exit points at Debug level for more granularity during development and}

\subsection*{testing.}

\subsection*{Debug}

\subsection*{This is the most verbose logging level (maximum volume setting).}

\subsection*{I usually consider Debug to be out-\/of-\/bounds for a production system and used it only for development and testing.}
